## 0. 总体架构（8 个功能共用的地基）

### 0.1 数据层：`assets/store.js`（新增）
浏览器 `localStorage` 模拟后端，**字段刻意对齐未来 Supabase 表**，将来换 `db.js` 即可平滑切换。统一管理：

| 数据 | 说明 |
|---|---|
| `user` | 当前登录用户（手机号、昵称、注册时间、邀请码、被谁邀请） |
| `vouchers` | 券池（用户已领的券 + 可用券） |
| `points` | 积分余额 + 流水（赚/花/过期） |
| `checkins` | 签到记录（连签天数、累计天数、最后签到日） |
| `orders` | 订单（现有 mock 迁进来，加 `paid/payMethod/voucherId/pointsEarned`） |
| `reviews` | 评价（绑定订单，初期仅后台可见） |
| `referrals` | 邀请关系 + 返积分记录 |
| `notifications` | 站内消息（模拟服务号推送） |

### 0.2 用户身份：手机号免验证
- H5 首次进入「会员中心 / 签到 / 用券」等需要身份的动作时，弹一个**手机号输入**（免验证码，MVP 占位），存进 `user`。
- 下单结算页的姓名/手机号自动带入。
- 预留升级位：将来可换成验证码登录或微信登录。

### 0.3 新增界面
- **H5 顾客端**（新增 screen）：`login` 登录、`me` 会员中心、`checkin` 签到、`pointsShop` 积分商城、`myVouchers` 我的券、`invite` 邀请好友、`messages` 消息中心、`review` 订单评价。底部 Tab 增加「我的」。
- **运营后台**（新增侧边栏分区）：券管理、积分/活动配置、订餐模式设置、评论反馈、裂变数据、推送日志。

## 1. 订餐用券（抵用券）

**券对象**：`{code, title, source, type, amount, threshold, scope, validFrom, validTo, status}`
- `source`：来源标记（`海螺升学返利` / `系统活动` / `邀请奖励` …），方便统计各业务板块发券与核销。
- `type`：`满减`（满 `threshold` 减 `amount`）/ `直减`（直接减 `amount`）/ `折扣`（打 X 折）。
- `scope`：`全场` / `指定餐厅` / `指定套餐`。
- `status`：`未使用` / `已使用` / `已过期`。

**H5**：结算页加「使用抵用券」入口 → 列出当前可用券（按门槛/有效期过滤）→ 选一张，订单金额实时显示抵扣。另支持「输入券码领取」。
**后台**：券管理页 —— 手动创建券、**批量导入**（外部业务如海螺升学把券推进来，先用 CSV/JSON 导入格式占位）、作废、核销记录查询。

> 现实依赖：真正与海螺升学打通需对方接口；本期做**自建券池 + 导入占位**，预留 `importVouchers()` 钩子。

## 2. 消费赠积分 + 兑换 + 定期清零

**赚积分**：订单**支付成功**后按实付金额赠送。
- ⚙️ **默认赚取比例**：实付 **$1 = 10 积分**（$15.99 ≈ 159 分，数字更有感）。
- **花积分**：积分商城兑换 —— 部分菜品/饮料、抵用券、活动抽奖券。
  - ⚙️ 示例兑换项：100 分换饮料券、300 分换 $5 抵用券、800 分换指定套餐券。
- **定期清零**：
  - ⚙️ **默认机制**：滚动 12 个月过期（积分按获得时间，满 12 个月自动过期）；会员中心显示「X 积分将于 MM/DD 过期」。
  - 备选：每年底 12/31 当年清零（更简单，但年底集中过期体验差）。

**UI**：会员中心顶部显示积分余额 + 积分流水；积分商城独立页。

## 3. 游戏化签到（提升打开率/粘性）

**每日签到**：每天可签 1 次，赠基础积分；连续签到有额外奖励，断签重置连击。
- ⚙️ **默认规则**：
  - 每日签到 **+5 分**；
  - **连签里程碑**：连续 7 天额外 **+50 分**，连续 15 天额外 **+120 分**（断签清零连击）；
  - **累计里程碑**：累计签到 30 天额外 **+200 分**（不受断签影响，长期累计）。

**UI**：会员中心「每日签到」—— 7 日横向日历、当前连签天数、明日奖励预告、「立即签到」按钮 + 签到动画/弹层。

## 4. 完善支付（占位模拟，支持 USD + 微信）

**支付方式**：结算页 `payMethod` 支持 **`card`（美元，Stripe 占位）** 和 **`wechat`（微信支付占位）**。
- 点支付 → 模拟「支付中…」→ 成功 → 生成订单 + 赠积分 + 核销券。
- `config.js` 增加 `STRIPE_PUBLISHABLE_KEY` / `WECHAT_PAY_*` 占位变量（空值，真实 key 待提供）。
- 预留 `pay(method, amount)` 钩子，将来 `card`→Stripe Checkout、`wechat`→微信支付 JSAPI/Native。

> 现实依赖：真实美元收款走 **Stripe**；真实微信支付需**商户号 + 合规主体**。本期只做闭环 mock。

## 5. 灵活订餐模式（后台可配）

现状：固定周一~周五整周。新增**后台开关**控制开放哪些天。
- ⚙️ **默认提供的模式**（待你确认这个集合）：
  - `整周`（周一~周五，现状）
  - `周一三五`
  - `周二四`
  - `自定义`（后台勾选开放哪几天）
  - `单天可订`（每天独立成单）
- H5 的日期选择器与 `buildMenu()` 只渲染**已开放**的天；截单时间随模式调整。

## 6. 订购后评论反馈（初期仅后台看，有凭据）

- **凭据**：评价**必须绑定一笔已完成订单**（防刷，证明真消费过）。⚙️ 是否**要求/允许上传照片**作为额外凭证？（默认：文字必填 + 照片选填）
- **评分维度**：餐厅评分 + 菜品评分（1–5 星）+ 文字。
- **可见性**：初期**只在后台展示**，前台不公开。
- **后台**：评论反馈页 —— 按餐厅聚合平均分、差评高亮、关联订单号，辅助决定**餐厅去留**。

## 7. 分享裂变（推广获客返积分）

- 每个用户有**专属邀请码/链接**。新用户经链接注册并**完成首单** → 双方返积分。
  - ⚙️ **默认奖励**：推荐人 **+200 分**，新人 **+100 分**（防刷：新人首单完成才结算）。
- 也支持把**菜单/套餐**分享卡片转发到微信（demo 用复制链接/生成分享卡模拟）。
- **UI**：会员中心「邀请好友」—— 我的邀请码、分享按钮、已邀请人数与返积分进度。
- **后台**：裂变数据 —— 邀请排行、转化与发积分统计。

---

## 8. 服务号/小程序提醒（取餐、邀评、奖励）

- **触发点**（设计钩子，本期站内模拟，真实接服务号后生效）：
  下单成功、截单前提醒、**备餐完成/可取餐**、订单完成后**邀请评价**、积分到账/即将过期、签到提醒、邀请成功。
- **本期实现**：站内「消息中心」+ 红点 + toast 模拟推送；统一 `notify(userId, type, payload)` 钩子。
- **后台**：推送日志（看哪些消息被触发）。

> 现实依赖：微信服务号模板消息需**已认证服务号**（主体资质 + 年费）；未来小程序需单独开发。本期只预留触发点与日志。
